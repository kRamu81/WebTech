# 📘 Lesson 1: Spring vs Spring Boot

## 🔹 Introduction

In Java development, both **Spring** and **Spring Boot** are widely used frameworks for building applications. While Spring provides the foundation, Spring Boot simplifies and accelerates development.

---

## 🌱 What is Spring?

**Spring** is a popular open-source framework that provides a comprehensive programming and configuration model for Java applications.

### ✅ Key Features:

* Supports **Dependency Injection (DI)**
* Helps in building **loosely coupled applications**
* Provides a wide range of **tools and libraries**
* Requires **manual server setup** (e.g., Tomcat)
* Needs **XML descriptors or configuration files**
* Involves **more boilerplate code**
* Requires **manual dependency management (pom.xml)**
* Does **not support in-memory database by default**

---

## 🚀 What is Spring Boot?

**Spring Boot** is built on top of Spring and is designed to simplify the development and deployment process.

### ✅ Key Features:

* Provides **Auto-Configuration**
* Supports building **standalone and enterprise applications**
* Comes with **embedded servers** (Tomcat, Jetty)
* No need for **XML configuration or descriptors**
* **Reduces boilerplate code**
* Supports **in-memory databases** like H2
* Automatically manages **dependencies**

---

## ⚖️ Spring vs Spring Boot

| Feature               | Spring Framework              | Spring Boot                     |
| --------------------- | ----------------------------- | ------------------------------- |
| Setup                 | Manual configuration required | Auto-configuration              |
| Server                | External server needed        | Embedded server (Tomcat, Jetty) |
| Configuration         | XML/Java-based configuration  | Minimal/no configuration        |
| Dependency Management | Manual (pom.xml)              | Automatic                       |
| Code Length           | More boilerplate              | Reduced code                    |
| Database Support      | No built-in in-memory DB      | Supports H2 and others          |
| Deployment            | Complex                       | Easy and fast                   |

---

## 🎯 Conclusion

* **Spring** gives full control and flexibility but requires more setup and configuration.
* **Spring Boot** makes development faster by reducing complexity and providing ready-to-use features.

👉 In modern development, **Spring Boot is preferred** for rapid application development.

---

## 📌 Summary

* Spring = Powerful but complex
* Spring Boot = Simplified and fast
